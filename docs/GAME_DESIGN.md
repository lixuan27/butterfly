# Butterfly（工作名）——游戏设计文档 v1

> **产品定位（用户 2026-07-17 晚指令）：最终产品 = 可交互游戏本体**，不是视频/不是纯 benchmark。
> 设计原则：玩法机制必须建立在**只有神经游戏才有的物理规则**上——混沌（噪声分叉）、有限记忆
> （0.85s 注意力窗）、时间线可分叉（.wsave）、梦境寿命（RoPE 164s 硬上限）。传统引擎无法复刻。

## 一句话

**你在一个由照片做成的梦里，与混沌对弈。**存档是你的锚，倒带是你的武器，蝴蝶效应是你的对手
（或你的画笔）。

## 世界观（把技术约束写成虚构）

- 你在清醒梦里。任何一张照片都能入梦（MG2 image-to-world）。
- 梦有**寿命**：360 个世界块（≈164 秒）后梦境坍塌——这是 RoPE 频率表的真实硬上限，不是人为设计。
- 梦是**混沌的**：完全相同的行为，世界也会在 ~4.4 秒内漂变成另一个版本（实测 T½=109.5 帧）。
- 你可以打**锚**（savepoint）：把整个梦的状态钉住，随时回去，或分裂成两条时间线。

## 模式

### 1. Anchor Duel「守梦」——旗舰模式（有技巧、有输赢、零先例）

**对抗混沌的收敛赛**：
1. 自由探索，任意时刻打锚，继续玩 T 个块（这段成为「原初时间线」，服务器记录帧+latent）
2. 发起 Duel：服务器回到锚点、**重掷噪声骰子**——混沌开始把世界推离原初时间线
3. 你重新操作这段时间，目标：**让你的世界与原初时间线保持一致**（屏幕左右并排：原初 ghost vs 你的世界；中间实时「相似度表」）
4. 结算：你的最终发散度 vs「无为基线」（M2 实测的纯噪声分叉曲线）→ 评级
   **Score = 你比"什么都不做"多守住了多少世界**（%），S/A/B/C 档

为什么有趣：混沌是真实的物理对手（不是脚本 AI）；你要学会读出世界要往哪漂、用镜头和走位把它拽回来。
为什么科学：这是**人在环的可控性测量**——人类玩家 vs 混沌的对抗数据直接进 SaveBench（M2 的人类上界）。

### 2. Butterfly「扇翅」——发散赛

同一锚点，你有 **k=1 次**改变动作的机会（其余块自动重放原动作）。目标：用最小的翅膀扇动造成
最大的世界分裂（结算 = 发散度/基线）。教学价值：亲手体验蝴蝶效应的放大动力学。

### 3. Relay「传梦」——异步多人

把 .wsave（2.7GB state 或 4.8MB reprime）发给朋友，他们在自己的时间线里续你的梦；时间线树
跨玩家生长。零 netcode 的多人。

### 4. Free「清醒梦」——沙盒

WASD+镜头自由探索 + 锚/倒带/分叉 + 时间线树 + 状态检查器（教学物）。

## HUD

- **梦境稳定度**：剩余块数倒计时（360 → 0，坍塌）
- **发散仪**：实时 latent 距离条（Duel 中 = 与 ghost 的距离；Free 中 = 与上一锚的漂移）
- **时间线树**：左侧栏，节点=锚，分支=分叉；点击=跳转
- **锚位**：3 个（逼迫取舍）

## 延迟设计（诚实面对 0.5s 块延迟）

动作以块为单位生效（12 帧≈0.5s）：**梦就是有惯性的**——设计语言上把延迟写成"梦的粘滞"，
玩法上所有模式都是策略/操控型而非反应型。帧流式播放 25fps 保证观感连续。

## 评分口径（Duel）

per-block latent MSE(玩家, ghost) 对时间平均 → d_player；基线 d_chaos = M2 纯噪声分叉同 horizon
的均值（实测曲线插值）。**Score = clamp(1 − d_player/d_chaos, −1, 1)**；≥0.35 S / ≥0.2 A / ≥0.05 B / else C。
（阈值待 playtest 校准；负分=你把世界推得比混沌还远，也会发生且好笑。）
